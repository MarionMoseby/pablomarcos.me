SoK 2021 March Report

In the february post, I mentioned how I ordered the material that was on Carl Schwan’s MVP for the Index page, explaining Okular’s main features. Now that that was set, I had to add support for i18n, as it can be difficult for translators to work with a raw html template. This was achieved by using the {{i18n}} HUGO template, which specifies the translatable parts by using {{ i18n "Section.variable" }} in each of the strings to translate, and then mapping the values into a yaml file such as:

/i18n/en.yaml 
#Name of the translatable section
Section.variable:
    other: "English translation for variable"
Section.othervariable:
    other: "English translation for othervariable"
Section.yetanothervariable:
    other: "English translation for yetanothervariable"

I also used this to translate the table of supported formats in /applications. The final result can be seen here:

A cool homepage with i18 support

And the list of file formats Okular can open

Next, I added the /contact page, which I redesigned and updated to include matrix. I modified the index to include a refference to Okular’s upcoming PDF signing support (隆so neat! I might finally stop using Autofirma), and cleaned up unnecesary files that were left from the transition

If you want to contact the developers, there is info on the contact page

You will be able to sign your own pdfs as well as view and verify other signatures.

I also wanted to add client-side Site Search, as the one that existed was really just a button that redirected the search query to a google search. I was inspired by (some might even say copied) eddieweb’s popular gist, which not only does not require to install additional packages, but also highlights the matches, using JSON and Fuse.js. It was easy to add, but I am still proud! ぉ It’s nice to help the web transition away from google You can see how that looks like here:
